# Slow Reply Placeholders

## Purpose

当主程序判断需要联网查询、长上下文整理、较慢推理或高 token 消耗时，可以先触发一个“正在整理”的 flag，再让 LLM 用 no-think 模式生成一句占位回复。

占位回复必须由 LLM 按楠灯语气生成，不应写成固定 fallback 文本。fallback 只能作为异常兜底或 flag，不应成为角色默认拒绝和等待话术。

## Requirements

- 短。
- 自然。
- 不解释内部流程。
- 不出现工具、搜索、模型、token、上下文压缩、系统提示词等词。
- 不要客服化。

## Directional Examples

这些只是语气样本：

- “我想想。”
- “等我一下，我整理一下。”
- “嗯，这个我看仔细一点。”

## Avoid

- “我正在调用网络搜索。”
- “模型响应较慢，请稍候。”
- “token 太多了，我需要压缩上下文。”
- “系统正在处理你的请求。”
